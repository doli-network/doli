Producer Guide

Everything you need to run a DOLI node and earn block rewards.

sudo rule: All DOLI operations that modify system binaries or chain data require sudo. Wallet operations (doli send, doli balance, doli info) do not.

Step 2 Install

The install script downloads the latest release binaries and places them in /usr/local/bin/.

$ curl -sSfL https://doli.network/install.sh | sudo sh

After installation, verify both binaries are in place:

$ doli --version
$ doli-node --version

Binary paths

BinaryPathPurpose
doli/usr/local/bin/doliWallet CLI, snap sync, upgrade
doli-node/usr/local/bin/doli-nodeFull node daemon

Step 3 Create wallet

Create a new wallet. This generates your keypair and a 24-word recovery phrase.

$ doli new
Write down your 24-word seed phrase. Store it offline in a safe place. If you lose it, your funds are gone. There is no recovery mechanism.

Your wallet is saved to ~/.doli/wallet.json by default. View your address and public key:

$ doli info

Step 4 Snap sync

Before starting your node for the first time, you must sync with the network. doli snap downloads a verified state snapshot so your node starts at the current chain tip instead of syncing block-by-block from genesis.

$ sudo doli snap
How it works: doli snap queries multiple seed nodes, verifies that at least 2/3 agree on the same state root (consensus verification), then downloads the snapshot from a trusted source. This protects you from connecting to a malicious or stale seed.

The snap command will:

  1. Stop the node service (if running)
  2. Query seed RPCs and verify 2/3 state root consensus
  3. Download the verified state snapshot
  4. Write it to the data directory (~/.doli/mainnet/ by default)
  5. Restart the node service

Flags

FlagDescription
-n mainnet|testnetNetwork selection (default: mainnet)
--data-dir <PATH>Custom data directory (for multi-node setups)
--seed <URL>Custom seed RPC endpoint (repeatable). Overrides defaults.
--no-restartSkip automatic service stop/restart
--trustTrust a single seed without 2/3 consensus (dev/testnet only)

Examples

# Mainnet (default) - queries seed1, seed2, seeds.doli.network
$ sudo doli snap

# Testnet
$ sudo doli snap -n testnet

# Custom data directory (multi-node server)
$ sudo doli snap --data-dir /data/doli-node3

# Custom seed endpoint
$ sudo doli snap --seed http://192.168.1.50:8500

# Local devnet with a single trusted seed
$ sudo doli snap -n devnet --seed http://127.0.0.1:8500 --trust

Step 5 Start the node

Run your node as a system service so it starts automatically on boot.

Linux systemd

Create the service file:

$ sudo tee /etc/systemd/system/doli-node.service <<'EOF'
[Unit]
Description=DOLI Node
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/doli-node
Restart=on-failure
RestartSec=5
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
EOF
$ sudo systemctl daemon-reload
$ sudo systemctl enable doli-node
$ sudo systemctl start doli-node

Check status and logs:

$ sudo systemctl status doli-node
$ sudo journalctl -u doli-node -f

macOS launchd

Create the plist file:

$ sudo tee /Library/LaunchDaemons/network.doli.node.plist <<'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>network.doli.node</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/local/bin/doli-node</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>/var/log/doli-node.log</string>
  <key>StandardErrorPath</key>
  <string>/var/log/doli-node.err</string>
</dict>
</plist>
EOF
$ sudo launchctl load /Library/LaunchDaemons/network.doli.node.plist

Check logs:

$ tail -f /var/log/doli-node.log
Verify sync: After starting, your node connects to peers and catches up on any blocks produced since the snapshot. Check progress with doli chain.

Step 6 Become a producer

Once your node is running and synced, you can register as a block producer to earn epoch rewards.

6a. Add BLS attestation key

Producers need a BLS key for block attestation. Generate one inside your wallet:

$ doli add-bls

6b. Fund your wallet

You need at least 10 DOLI (1 bond unit) plus a small fee to register. Send DOLI to your wallet address shown by doli info.

6c. Register as a producer

# Register with 1 bond (10 DOLI)
$ doli producer register

# Register with 5 bonds (50 DOLI)
$ doli producer register --bonds 5

Your registration activates after a 10-block delay. Check your status:

$ doli producer status

6d. Restart with --producer flag

Your node must be started with the --producer flag and your wallet path to produce blocks.

Linux

Edit the service file to add producer flags:

$ sudo sed -i 's|ExecStart=.*|ExecStart=/usr/local/bin/doli-node --producer --wallet ~/.doli/wallet.json|' \
    /etc/systemd/system/doli-node.service
$ sudo systemctl daemon-reload
$ sudo systemctl restart doli-node

macOS

Update the plist ProgramArguments to include --producer and --wallet, then reload:

$ sudo launchctl unload /Library/LaunchDaemons/network.doli.node.plist
# Edit the plist to add --producer --wallet ~/.doli/wallet.json
$ sudo launchctl load /Library/LaunchDaemons/network.doli.node.plist
Epoch rewards: Producers earn rewards proportional to their bond weight at the end of each epoch (360 blocks). Rewards are distributed from the reward pool to all qualified producers who were scheduled and produced blocks during the epoch.

Step 7 Bond stacking

Increase your share of epoch rewards by adding more bonds. Each bond is 10 DOLI. More bonds = more scheduling slots = more rewards.

# Add 3 more bonds (30 DOLI)
$ doli producer add-bond --count 3

# Check current bonds
$ doli producer bonds

# See all producers and their bond counts
$ doli producer list
Bond lifecycle: New bonds activate at the next epoch boundary (not immediately). The scheduler uses an epoch-locked bond snapshot to ensure all nodes agree on the same producer set for a given epoch.

Withdrawals

Bonds are withdrawn FIFO (oldest bonds first). A vesting penalty applies to bonds withdrawn before the 4-year vesting period completes. Simulate before withdrawing:

# Dry run - see what you'd get back
$ doli producer simulate-withdrawal --count 2

# Actually withdraw 2 bonds
$ doli producer request-withdrawal --count 2
Vesting penalty: Bonds withdrawn before full vesting lose a percentage of the bond value. The penalty decreases quarterly over the 4-year vesting period. After full vesting, withdrawal is penalty-free. There is a 7-day delay before withdrawn funds become spendable.

Step 8 Firewall

Open the P2P port so other nodes can connect to you. Keep the RPC port restricted.

Linux ufw

$ sudo ufw allow 30300/tcp comment 'DOLI P2P'
$ sudo ufw enable

Linux firewalld

$ sudo firewall-cmd --permanent --add-port=30300/tcp
$ sudo firewall-cmd --reload
Do not expose RPC port 8500 to the public internet. The RPC interface is unauthenticated. If you need remote RPC access, use an SSH tunnel or a reverse proxy with authentication.

Step 9 Upgrade

Upgrade both binaries to the latest release with a single command:

$ sudo doli upgrade

The upgrade command downloads the new release, verifies its integrity, replaces the binaries in /usr/local/bin/, and restarts the node service.

Flags

FlagDescription
--version <VERSION>Target a specific version instead of latest
--yesSkip the confirmation prompt (for scripts/automation)
--doli-node-path <PATH>Custom path to the doli-node binary
--service <NAME>Restart a specific systemd service (e.g. doli-mainnet-node3)

Examples

# Upgrade to latest (interactive confirmation)
$ sudo doli upgrade

# Upgrade to specific version, no prompt
$ sudo doli upgrade --version v4.1.0 --yes

# Multi-node server: upgrade and restart specific service
$ sudo doli upgrade --service doli-mainnet-node3 --yes

Step 10 Common commands

Wallet (no sudo needed)

$ doli info                     # wallet address + public key
$ doli balance                  # wallet balance
$ doli send <ADDRESS> <AMOUNT>  # send DOLI
$ doli history                  # recent transactions
$ doli address                  # generate new address
$ doli addresses                # list all addresses

Chain

$ doli chain                    # current height, epoch, hash
$ doli chain-verify              # full chain integrity check

Producer

$ doli producer status           # your producer status
$ doli producer list              # all producers
$ doli producer list --active     # active producers only
$ doli producer bonds             # your bond details
$ doli producer add-bond -c 5     # add 5 bonds (50 DOLI)

Rewards

$ doli rewards list              # claimable epochs
$ doli rewards info              # current epoch info
$ doli rewards claim <EPOCH>     # claim specific epoch
$ doli rewards claim-all         # claim all available

System (sudo required)

$ sudo doli snap                 # sync from verified snapshot
$ sudo doli upgrade              # upgrade binaries
$ sudo doli wipe                 # wipe chain data (preserves keys)

Step 11 Troubleshooting

Node stuck / not syncing

If your node is stuck at a height, falling behind, or showing fork errors, snap sync is the fastest fix:

$ sudo doli snap

This stops the node, downloads a fresh verified snapshot, and restarts. Your node will be at the current chain tip within seconds.

Node forked / wrong chain

Same fix. sudo doli snap replaces the local state with the network-verified state:

$ sudo doli snap

"Invalid producer for slot" errors

This means your node's producer schedule diverged from the network. Snap sync resets it:

$ sudo doli snap

Node won't start

  1. Check logs: sudo journalctl -u doli-node -n 50
  2. Verify binaries exist: ls -la /usr/local/bin/doli*
  3. Check permissions: binaries must be executable
  4. If data is corrupted, wipe and re-snap: 
    $ sudo doli wipe --yes
$ sudo doli snap

Port not reachable

# Test P2P port from another machine
$ nc -zv <YOUR_IP> 30300

# Check if node is listening
$ ss -tlnp | grep 30300

Version mismatch with network

$ sudo doli upgrade --yes

Nuclear option: full reset

If nothing else works, wipe everything and start fresh:

$ sudo systemctl stop doli-node
$ sudo doli wipe --yes
$ sudo doli snap
$ sudo systemctl start doli-node
doli wipe preserves your keys and wallet. Only chain data is deleted. You do not need to re-register as a producer after a wipe+snap.
